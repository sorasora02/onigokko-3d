# onigokko-3d

## 概要

Unityで制作した3D鬼ごっこゲームです。
プレイヤーはTPS視点でステージ内を移動し、追いかけてくる鬼から逃げながらコインを集めます。
制限時間まで逃げ切り、かつ難易度ごとに決められた必要スコアを達成するとクリアになります。

## ゲームルール

- ゲーム開始時に Easy / Normal / Hard から難易度を選択します。
- 難易度によって、鬼の移動速度とクリアに必要なスコアが変わります。
- 鬼に捕まるとゲームオーバーです。
- 制限時間まで逃げ切っても、必要スコアに届いていない場合はゲームオーバーです。
- コインを取るとスコアが増えます。
- コインの上には、そのコインが何点か分かるテキストを表示しています。

## 操作方法

| 操作 | 内容 |
| --- | --- |
| W / A / S / D | 移動 |
| マウス移動 | 視点操作 |
| R | リセットして難易度選択に戻る |
| Escape | 終了確認を表示 |
| Y | 終了確認中にゲームを終了 |
| N | 終了確認をキャンセル |

## 使用技術

- Unity
- C#
- Unity Input System
- Unity NavMeshAgent
- TextMeshPro

## 実装した主な機能

- TPS視点のカメラ
- カメラの向きを基準にしたプレイヤー移動
- NavMeshAgentを使った鬼の追跡
- Easy / Normal / Hard の難易度選択
- 難易度ごとの鬼の速度変更
- 難易度ごとのクリア必要スコア変更
- コイン取得によるスコア加算
- コイン上の点数表示
- 制限時間とスコアによるクリア判定
- 鬼との接触によるゲームオーバー判定
- リセット機能
- Y / N キーによる終了確認
- 操作説明UI
- 最終スコア表示

## 工夫した点

### カメラ基準の移動

Wキーを押したときにワールド座標の前方向へ進むのではなく、カメラから見た前方向へ進むようにしました。
これにより、TPSゲームとして直感的に操作できるようにしています。

### 難易度選択

Easy / Normal / Hard の3種類を用意し、鬼の速度とクリアに必要なスコアを変えました。
同じステージでも、難易度によって逃げやすさとスコア取得のリスクが変わるようにしています。

### スコア条件付きの鬼ごっこ

時間いっぱい逃げ切るだけではなく、必要スコアを集めることもクリア条件にしました。
安全に逃げるか、危険を承知でコインを取りに行くかという判断が生まれるようにしています。

### 見やすいコイン表示

コインの上に点数を表示し、どのコインを取ると何点入るのか分かりやすくしました。
また、点数表示が読みにくくならないように、コイン本体だけを回転させる構造にしています。

### 3Dステージ

壁や高低差のあるステージにすることで、3D空間を使った逃走ルートを作れるようにしました。
平面の鬼ごっこではなく、障害物や地形を利用して逃げるゲーム性を意識しています。

## 制作目的

3Dゲームまたは3Dグラフィックスを用いたゲームとして提出するために制作しました。
短期間で完成させることを優先し、基本的なゲームループである「難易度選択、操作、逃走、スコア取得、クリア・ゲームオーバー判定」が成立する形を目指しました。

## 実行ファイルについて

Unityのビルドは、exeファイル単体では動作しません。
提出用の実行ファイルとして渡す場合は、ビルドで生成されたフォルダ全体をzipにして提出します。

例:

```text
onigokko-3d-build.zip
```

zipの中には、exeファイルのほかに、DataフォルダやUnityPlayer.dllなどの実行に必要なファイルが含まれている必要があります。
